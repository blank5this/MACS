# 面试评分表 (自评用) — ERP AI Copilot 项目

> 适用: 1-3 年经验 AI 应用工程师 / Python 后端候选人自评
> 目的: 投递前 1 周, 客观评估自己**当前水平 + 6 个月目标 + 2 年目标**
> 用法: 每维度打分 (L1-L5), 写出自评矩阵, 列 3 个 3 个月内可改进的目标.

## 二、技术深度 (5 个等级)

### L1 — 入职级 (应届 / 0-1 年)
- 知道 LLM / Agent / RAG 是什么, 能跑通 Demo
- 代码能跑, 但不会考虑**生产级** (timeout / retry / 错误处理)
- 业务理解停留在"调 API"
- 典型特征: 简历写"熟悉 LLM", 但被问"用哪个 Provider"答不上来

### L2 — 胜任级 (1-3 年, **我的目标 Lv**)
- 独立完成**端到端产品** (数据 + Agent + 检索 + 部署)
- 至少**实测**过单 Agent vs 多 Agent, 能讲清楚为什么选
- 理解 LLM 成本 / 不稳定性 / 失败模式, 有 mitigation
- 写过**生产级**代码 (CI / 测试 / 健康检查 / 错误传播)
- 典型特征: 168 测试金字塔, 4 层 SQL 防护, 演示脚本内置 Scripted LLM

### L3 — 高级 (3-5 年)
- 设计**跨团队** AI 架构 (统一 LLM Provider 抽象 / RAG 平台化)
- 深度调优 LLM (prompt 调 3+ 版本, few-shot, chain-of-thought)
- 跑过**大规模** (10K+ QPS / 100K+ 文档 RAG)
- 主导过 AI 项目**从 0 到 1**, 带过 1-2 个初级工程师
- 典型特征: 简历有"主导"字样, 面试能讲"我做的选型, 团队沿用 1 年"

### L4 — 资深 (5-8 年)
- 设计**公司级** AI 平台 / 框架
- 深度优化**成本** (token / 推理 / 缓存) — 把成本压到同行 50% 以下
- 主导过**百万级营收** AI 项目
- 写过**行业文章 / 演讲**, 在社区有 reputation
- 典型特征: LinkedIn 500+ 关注, GitHub 1K+ star

### L5 — 专家 (8+ 年)
- 定义**行业标准** (LangChain 核心 contributor 级别)
- 主导过**跨公司** AI 标准制定
- 团队规模 20+, 培养过多位 L4
- 典型特征: 业界公认的"AI 架构师", 经常被邀请 talk

### 我目前在 L2 上半段 (1-3 年胜任)
- 证据: ERP AI Copilot 项目覆盖 L2 全部 5 条标准
- 缺口: 没跑过 10K+ QPS, 没带过人, 没在 1K+ 文档 RAG 上压测过

## 三、业务理解 (5 个等级)

### L1 — 入职级
- 知道"AI 能做", 不知道"AI 不能做"
- 不会区分**业务价值** vs **技术 demo**
- 典型问题: "我想做个通用 AI 助手, 能聊天能写代码能画画"

### L2 — 胜任级 (我的目标 Lv)
- 知道 ERP / 金融 / 电商 各自 AI 落地的**真实痛点**
- 能讲清楚 "我做的 AI 解决了 ERP 哪 3 个具体业务问题"
- 理解**业务方采纳率**比准确率更重要
- 典型特征: 风险评分公式 / MOQ 政策 / Net 30 付款条款 这些业务概念熟悉

### L3 — 高级
- 主导过 **业务方 + 工程师** 联合工作坊, 翻译需求
- 设计过 **业务方反馈循环** (采纳率 / 不采纳率 → 训练数据)
- 业务方找你**问 AI 能不能做 X**, 不是你问业务方

### L4 — 资深
- 主导过 **行业级** AI 落地 (e.g. 全国 100+ 医院部署医疗 AI 助手)
- 写过 **行业白皮书**
- 给业务方做 **AI 培训**

### L5 — 专家
- **定义行业** AI 标准 (医疗 AI 准入 / 金融 AI 合规)
- 政府 / 行业协会 顾问

### 我目前在 L2 中段
- 证据: 18 篇中文 ERP 制度文档 + 风险评分公式 + 5 个 Plan 外项
- 缺口: 没和真实业务方对接过 (项目是 simulated data), 没做过业务方培训

## 四、沟通表达 (5 个等级)

### L1 — 入职级
- 自我介绍 3 分钟讲不到重点
- 技术问题回答**没结构** (讲 5 分钟没结论)
- 演示**紧张**, 跑挂就慌

### L2 — 胜任级 (我的目标 Lv)
- 自我介绍有 **3 种长度** (1/3/5 分钟), 知道什么时候用哪个
- 技术问题回答有 **"结论 + 证据 + 数字"** 三段式
- 演示有 **剧本** (30 分钟分 5 步, 应急方案准备好)
- 典型特征: 录音回放没口头禅, 5 分钟版计时 5:30 封顶

### L3 — 高级
- 能在 **50 人技术分享** 讲 1 小时, 听众不困
- 写 **技术博客** 1K+ 赞
- 演示时**带节奏**, 知道哪里该停, 哪里该快

### L4 — 资深
- 能在 **行业大会** 演讲 (QCon / ArchSummit)
- **带人** 表达, 1 对 1 mentor 初级能快速成长
- 写 **书** / **专栏**

### L5 — 专家
- **定义行业话语** (e.g. "Transformer 杀手" 这种 meme 制造者)
- TED 演讲级别

### 我目前在 L2 中段 (有结构, 但演练不够)
- 证据: 3 种长度自我介绍 + 5 步演示剧本 + 24 题反问清单
- 缺口: 没在 50 人面前讲过, 没写过技术博客

## 五、抗压能力 (5 个等级)

### L1 — 入职级
- 演示挂了就**僵住**, 不会用应急方案
- 刁钻题答不上来**编答案**, 越编越乱
- 面试官追问**改口**, 显得没底气

### L2 — 胜任级 (我的目标 Lv)
- 演示挂了有 **3 个应急方案** (Postgres / API / 编码)
- 刁钻题答不上来**诚实说** "这块我没经验, 我讲讲我的理解"
- 面试官追问**承认细节** "这部分我用的是 X, 没试过 Y, 但理论应该..."
- 典型特征: 失败案例能讲 1 分钟不甩锅

### L3 — 高级
- 演示挂了**自己 debug**, 面试官看着你排错
- 刁钻题能**反问回去** "您问的是不是 X? 我更熟悉 Y"
- 节奏控制好, **主动叫停** "这点我们 5 分钟聊不完, 留到二面"

### L4 — 资深
- 演示挂了能**自嘲** "您看, 这就是我没做的兜底, 正好现场补上"
- 刁钻题能**引导** "您的提问让我意识到, 这块我应该再深挖"

### L5 — 专家
- 任何刁钻题都能**接住** + **反推** 1-2 个新洞察
- 面试变成**平等的交流**, 不是 Q&A

### 我目前在 L2 上半段
- 证据: 演示应急方案 (Postgres 没起 / API 超时 / GBK 编码) + 失败案例 (v1.0.0 吞异常) 准备
- 缺口: 没经历过真实面试刁钻, 演练次数有限

## 七、"面试官眼中"的我 (10 个常见评价, 正反各 5)

> 这是面试结束后面试官可能给的 5 个正面 + 5 个负面评价, 提前**自我审视**.

### 正面评价 (我期望听到的)

1. **"项目完整度高"** — "他不只是写了代码, 还做了 CI / 健康检查 / 演示脚本, 像一个有产品意识的工程师"
   - 证据: 168 测试 + 4 层 SQL 防护 + 3 维健康检查 + 3 段视频脚本

2. **"业务理解深"** — "他不只讲技术, 能讲清楚 AI 解决了 ERP 哪 3 个具体痛点"
   - 证据: 风险评分公式 / MOQ 政策 / 3 个 ERP 痛点 (库存 / 采购 / 制度)

3. **"反思能力强"** — "他讲了一个 v1.0.0 吞异常的 bug, 而且讲了 v1.0.1 怎么修的, 不是甩锅"
   - 证据: `ROADMAP_AUDIT_v1.0.1.md:131-151` 失败案例

4. **"演示准备充分"** — "他有 3 段 60s 视频脚本, 还有 30 分钟演示剧本, 明显是练过的"
   - 证据: `docs/videos/` 3 段脚本 + `INTERVIEW_TECH_DEMO.md`

5. **"知道自己不知道"** — "他诚实说 RAG 没用过 1000+ 文档, 没在大厂跑过 Agent, 但愿意学"
   - 证据: `INTERVIEW_QA_BANK.md` T5 — "1-3 年定位是诚实评估, 不夸大"

### 负面评价 (我要规避的)

1. **"技术深度不够"** — "他讲的都是他做过的, 问他 LLM fine-tune / 分布式 Agent 一问三不知"
   - 规避: 在 T5 主动承认短板, 提"愿意学"

2. **"业务理解浅"** — "他做的项目是 simulated data, 没和真实业务方对接过"
   - 规避: 主动说"数据是 Faker 模拟, 但业务概念 (MOQ / Net 30) 我是熟悉的"

3. **"演示紧张"** — "他中间卡壳了 5 秒, 然后跳过了一段"
   - 规避: 演示前完整跑 1 遍, 计时 30 分钟

4. **"回答太长"** — "他 1 个问题答了 5 分钟, 我没耐心听完"
   - 规避: 答案控制在 1-2 分钟, 主动说"如果您想深挖可以展开"

5. **"反问像背 checklist"** — "他问的'加班频率 / 调薪周期' 像是从网上抄的"
   - 规避: 反问用"团队目前 AI 落地到什么阶段" 这种**业务向**问题, 不用"加班打车费" 这种**福利向**问题

## 十、附录 — "挂掉的 5 个信号"

> 面试官可能没明说, 但这 5 个信号意味着大概率挂.

| 信号 | 表现 | 规避 |
|------|------|------|
| 1. 一直打断你 | "这个我们不需要 / 这个我了解了" | 答案控制在 1-2 分钟, 主动说"如果您想深挖可以展开" |
| 2. 看手机 / 电脑 | 不看你的眼睛 | 不必慌张, 保持节奏, 用演示转移注意力 |
| 3. 反问特别敷衍 | "没什么想问的" | 主动分享 1 个细节 (你项目里最好玩的技术), 拉回节奏 |
| 4. 提薪资时支吾 | "我们再看看" | 礼貌追问 "薪资范围大概多少? 我看是否合适再继续" |
| 5. 5 分钟内结束 | 1 轮 30 分钟面试 10 分钟就收 | 主动展示 1 个亮点 (NL→SQL 4 层防护 demo), 拉长节奏 |

---

> **最后一句**: 自评不是"自卑", 是"知己". L2 中段不是失败 — 1-3 年经验定位 L2 中段, 拿 15-25K offer 完全合理. 关键是** 3 个月内把 L2 中提到 L2 上**, 那是面试官眼里"超额完成"的候选人.
