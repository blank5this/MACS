### API

```python
from macs_pkg import CitationTracker

tracker = CitationTracker()

# 1. 注册来源
l1, _ = tracker.track(url="https://...", title="论文", snippet="关键段落")

# 2. 绑定声明
claim_id = tracker.bind_claim(
    source_ids=[l1],
    claim_content="X 已被证明",
    turn=1,
)

# 3. 格式化报告（含 [1][2] 编号 + 参考文献脚注）
report = tracker.format_report(raw_text, inline=True, footnotes=True)

# 4. 完整引用图
graph = tracker.get_citation_graph()
```

### ReviewerAgent 集成

## 二、P0-1 DeepResearchMode — 深度研究编排循环（已完成 ✅）

**动机：** OpenAI Deep Research、Manus 的核心差异化在于「多轮检索→反思→综合」的明确循环。STORM 论文（arXiv:2410.09663）是标杆参考架构。

### 新增文件

| 文件 | 说明 |
|------|------|
| `macs_pkg/collaboration/deep_research.py` | 核心模块（约 500 行）：5 个数据类 + `DeepResearchMode` 引擎 |
| `tests/test_deep_research.py` | 20 个测试，覆盖所有方法 |
| `macs_pkg/collaboration/__init__.py` | 新增导出 |

### 架构

```
execute() ── loop iter ──────────────────────
  ├─ Phase1: _plan_queries()        PlannerAgent LLM 分解 或 启发式 fallback
  ├─ Phase2: _parallel_search()      asyncio.gather 并行执行所有搜索+RAG 工具
  │       ├─ _search_single_query()
  │       ├─ _parse_search_result()    → SearchResultOutput
  │       └─ _parse_rag_result()
  ├─ Phase3: _extract_claims()      按 query_id 分组证据 + 绑定 CitationTracker
  ├─ Phase4: _synthesize_report()  调用 ReviewerAgent.synthesize() 生成结构化报告
  └─ Phase5: _self_critique()       质量评分 < threshold → 修订 topic → 下一轮
```

### 使用示例

```python
from macs_pkg import DeepResearchMode, CollaborationConfig
from macs_pkg.tools.web_search import DuckDuckGoSearchTool

mode = DeepResearchMode(
    config=CollaborationConfig(max_iterations=3),
    quality_threshold=0.75,
)

report = await mode.execute(
    task={"topic": "量子计算在金融领域的应用"},
    agents={"planner": planner, "reviewer": reviewer},
    context={
        "search_tools": [DuckDuckGoSearchTool()],
        "rag_tools": [rag_tool],
    },
)

print(report.title)        # "量子计算在金融领域的应用"
print(report.sections)    # [{title, content}, ...]
print(report.citations)   # [{url, title, snippet}, ...]
print(report.quality_score)  # 0.85
```

## 三、P0-2 ReactAgent — think/act 生命周期解耦（已完成 ✅）

**动机：** 原有 `think()` 和 `act()` 独立可调用，漏掉一步则 LLM 不触发。这是已知的脆弱设计。SOTA 的 ReAct/PlanRAG 框架均明确分离「规划」和「执行」阶段。

### 新增文件

| 文件 | 说明 |
|------|------|
| `macs_pkg/core/react_agent.py` | `ReactAgent` 基类（~100 行）+ RuntimeError 强制校验 |
| `tests/test_react_agent.py` | 8 个测试，覆盖所有边界情况 |

### 核心保障

| 场景 | 行为 |
|------|------|
| `act()` 先于 `think()` | `RuntimeError: act() called before think()` |
| `think()` 重复并发 | `RuntimeError: think() already in THINKING state` |
| `act()` 重复并发 | `RuntimeError: act() already in ACTING state` |
| `run()` | 一句完成完整 cycle，自动重置 flag |

### 使用

```python
from macs_pkg.core import ReactAgent, AgentRole, Message

class MyAgent(ReactAgent):
    async def _think_impl(self, msg):
        return Message(...)  # LLM 规划

    async def _act_impl(self, response):
        return [result_msg]  # 工具调用 / 消息发送

agent = MyAgent(name="assistant")

# 正确用法
await agent.run(msg)

# 或手动（强制顺序）
response = await agent.think(msg)
await agent.act(response)  # RuntimeError if think() skipped
```

## 四、P0-3 工具结果 Pydantic Schema 化（已完成 ✅）

**动机：** SOTA agent 均强制 tool output schema（Pydantic model），便于 LLM 可靠解析、跨 agent 传递、自动评估。原有 `ToolResult` 是非结构化字符串。

### 改动文件

| 文件 | 改动 |
|------|------|
| `macs_pkg/tools/base.py` | 完全重构：新增 `SearchResultOutput`（pydantic）、`TypedToolResult`、向后兼容别名 |
| `macs_pkg/tools/web_search.py` | 全面重写：`_search()` 返回 `List[SearchResultOutput]`，`run()` 同时返回格式化字符串 + `metadata["items"]` 结构化数组 |
| `macs_pkg/tools/__init__.py` | 导出 `TypedToolResult`、`SearchResultOutput` |

### 新增类型

| 类型 | 说明 |
|------|------|
| `SearchResultOutput`（pydantic） | `url / title / snippet / source / score` + `to_citation_dict()` |
| `TypedToolResult` | 带 `items: List[Any]` 的工具结果；`.first()` / `.to_tool_result()` |
| `SearchResult` | `SearchResultOutput` 的向后兼容别名（已有代码无需改动） |

### 搜索工具返回值结构

```python
result = await search_tool.run("量子计算")
# result.output        格式化字符串 "[1] Title\n   URL: ...\n   snippet"
# result.metadata["items"] 结构化列表
# result.metadata["query"]
```
