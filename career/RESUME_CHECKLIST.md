## 一、简历本体检查 (10 项)

> 文件: `career/RESUME.md` → 输出 PDF

- [ ] **1. 字数 ≤ 1 页 A4**: 全文 ~800 字, 打印 PDF 字号 10pt / 行距 1.15 / 边距 1.5cm, 不超 1 页
- [ ] **2. 个人简介 80-100 字**: "23 岁 Java 后端 1-3 年, 转型 AI 应用工程师 15 天完成 ERP AI Copilot v1.0.1", 不超 100 字, 不低于 80 字
- [ ] **3. 项目经历 4 个 bullet**: ERP AI Copilot 项目展开 4 个 bullet (数据层 / Agent 层 / RAG 层 / 工程化), 每个 1 行 ≤ 50 字
- [ ] **4. 技能清单 3 列**: 编程语言 / 框架与工具 / AI·LLM 相关, 每列 5-6 项, 不超过 18 项总数
- [ ] **5. 技术栈全**: Python 3.11 + FastAPI + PostgreSQL + AutoGen/LangChain + MCP + Docker + GitHub Actions, 7 项不少
- [ ] **6. 数字具体**: 168 测试 / 22 文件 / 18 KB / 5 表 / 1000 行 / 4 层防护 / 6 LLM Provider, 数字跟 ROADMAP_AUDIT_v1.0.1.md 对齐
- [ ] **7. GitHub 链接**: github.com/blank5this/MACS, 在头部 + 项目经历各出现 1 次, 链接可点
- [ ] **8. 无错别字**: "开发" 不能写成 "开必", "部署" 不能写成 "布署", 通读 2 遍
- [ ] **9. 无 AI 八股**: 不写"在当今 AI 浪潮下" / "期待与贵司一起探索 AI 无限可能" / "我是一个有责任心有担当有热情的工程师", 这些都是减分项
- [ ] **10. PDF 格式**: 输出 PDF 后用 Adobe Reader 打开, 检查链接可点 / 字体不乱码 / 中文不方块

## 四、投递前 1 小时必做 (5 项)

> 收到 HR 回复 / 主动投递前最后一遍检查

- [ ] **1. 读岗位 JD 至少 5 分钟**: 提取 3-5 个关键词 (Python / FastAPI / RAG / Agent / LLM), 写到本子或简历旁边, 准备匹配
- [ ] **2. 改 1-2 个 bullet 匹配 JD**: 如果 JD 强调 "RAG", 简历项目经历里把 RAG 那条 bullet 调到最前; 如果 JD 强调 "Agent", 调到最前. 不要全改, 只调顺序
- [ ] **3. 准备 1 个反问面试官**: "团队目前 AI 应用落地到什么阶段?" / "这个岗位未来 3 个月最希望我解决什么问题?" / "Agent 框架是自研还是用 LangChain/AutoGen?", 至少准备 1 个
- [ ] **4. 看公司近期新闻**: 打开"36 氪 / 虎嗅 / 公司官网 News" 看 1-2 篇最新文章, 准备 1 句"我看到贵司最近做了 XXX, 这跟我的项目经验很契合"
- [ ] **5. 准备自我介绍 3 种长度**:
  - 30 秒版 (1 句话): "我是 <your_name>, 做了 ERP AI Copilot v1.0.1, 应聘 AI 应用工程师岗."
  - 2 分钟版 (项目 1 段): 用 RESUME_PROJECT_DESCRIPTION.md 第 2 版本
  - 5 分钟版 (面试开场白): 用 INTERVIEW_OPENING.md, 演练 ≥ 3 遍

## 五、面试前 1 小时必做 (5 项)

> 收到面试邀约后, 出门前 1 小时最后检查

- [ ] **1. 复习 5 个面试高频题答案**: 用 PROJECT_PITCH.md 第 7 节
  - Q1: 为什么使用多 Agent? (3 句: 单 Agent prompt 膨胀 / 拆 Agent 后成功率 70% → 95% / 职责分离)
  - Q2: 为什么不用单 Agent? (2 句: 实际试过 / 关注点分离)
  - Q3: 如何控制成本? (3 层: Context 裁剪 / Agent 分离 / Mock LLM)
  - Q4: 如何处理 Agent 失败? (3 道防线: Retry / Timeout / Fallback + v1.0.1 修了 error propagation)
  - Q5: 和 LangChain/AutoGen 区别? (3 点: 业务价值 / 可演示 / 可工程化)
- [ ] **2. 复习 5 个超出 Plan 项**: 用 ROADMAP_AUDIT_v1.0.1.md 第 2 节
  - Web UI (Day 12, 给非技术同事用)
  - CI/CD 独立 workflow (Day 13, 4 job: lint/unit/integration/coverage)
  - 3 维健康检查 (Day 13, DB/LLM/RAG, 单一事实源)
  - SQL 4 层防护 (Day 6, AST/黑名单/白名单/参数化)
  - 168 测试金字塔 (Day 1-13, 168 单元 + 23 集成 + 6 e2e)
- [ ] **3. 准备 3 个反问面试官的问题**:
  - "团队目前 AI 应用落地到什么阶段? 已经上线还是 POC?"
  - "这个岗位未来 3 个月最希望我解决什么问题? 是搭框架还是做业务?"
  - "Agent 框架选型是自研还是基于 LangChain/AutoGen? 我可以基于现有 MACS 框架快速适配."
- [ ] **4. 准备 1 个杀手锏演示**: 现场打开 GitHub 仓库 + Web UI, 跑 1 个真实 query (如"哪些商品库存低于安全库存?"), 让面试官看 Planner → Analyst → Buyer → Writer 4 Agent 协作输出 Markdown 报告
- [ ] **5. 复习 5 个兜底答案**: 用 PROJECT_PITCH.md 第 11 节"反向思考"
  - "你的项目和大厂 Copilot 区别?" — 重点不是替代大厂, 是证明 15 天能搭出端到端产品
  - "SQL 4 层防护能挡住所有注入吗?" — 覆盖已知风险, 未知风险需加 LLM result 校验
  - "为什么不用 LangGraph/AutoGen GroupChat?" — 我的场景是 4 Agent 串行, Hierarchical 更直接

## 八、红线检查 (绝对不能犯)

- [ ] **不能造假**: Java 工作经历年限 / 学历 / 公司不要造假, 面试背景调查会被查出来
- [ ] **不能群发**: 求职信 / Boss 打招呼必须改公司名 + 岗位名, 不能"您好, 应聘贵司 AI 工程师岗"
- [ ] **不能写错数字**: 168 测试 / 5 张表 / 18 KB / 1000 行 / 4 层防护, 这些数字必须跟 ROADMAP_AUDIT_v1.0.1.md 对齐, 不许改
- [ ] **不能 emoji**: 简历 / 求职信 / Boss 打招呼, 全部不用 emoji
- [ ] **不能写面议**: 薪资必须写 15-25K, 不写"面议" — HR 会按 12K 给你开
- [ ] **不能漏 GitHub**: 简历 / 求职信 / Boss 打招呼, GitHub 链接必须出现 ≥ 1 次
- [ ] **不能背稿感**: 面试开场白要口语化, 用 "·" 表示自然停顿, 不要一字不差背诵

## 九、紧急联系方式 (面试突发状况)

| 状况 | 应对 |
|------|------|
| 面试官问 Java 经验, 我 Java 不强 | "Java 我做了 1-3 年 Spring Boot, 但过去 15 天我把精力放在 AI 应用层 (ERP AI Copilot), 这个项目更对标贵司 JD. Java 工程基础 (Spring Cloud / MySQL / 设计模式) 我可以快速迁移, 这是 Java 转 AI 应用工程师的常见路径." |
| 面试官问 "你为什么离开上家?" | "我想从 Java 后端转型 AI 应用工程师, 我用 15 天做了完整项目证明能力, 现在准备全身投入 AI 方向. 这不是逃离, 是主动选择." |
| 面试官说 "我们要求 3 年 AI 经验" | "我没有 3 年 AI 工作经验, 但我有的: 1 个完整上线的 AI 应用产品 (ERP AI Copilot v1.0.1) + 168 测试 + 4 层 SQL 防护 + 多 Agent + RAG. 这些工程化能力比 3 年零散 demo 更有说服力, 您看能否给个面试机会让我现场演示?" |
| 面试官说 "我们薪资给不到 25K" | "理解, 我的薪资范围是 15-25K, 20K 是我的期望值, 15-18K 是起步线. 如果 base 18K + 期权 + 年终 3 个月, 总包能到 25K, 我也能接受." |
